# /blogkit.specify Command

**Purpose**: Create blog content strategy specification without technology implementation details.

**Usage**: Run `/blogkit.specify` in Claude Code to define blog content strategy (audience, pillars, success metrics).

**Status**: PLACEHOLDER - Full implementation in Phase 5 (T035-T041)

This slash command will:
1. Create new feature branch (e.g., `001-authority-blog-ml-engineers`)
2. Generate `specs/NNN-feature-name/spec.md` using blog-specific spec template
3. Enforce Content-First principle (reject CMS, framework, hosting mentions in spec phase)
4. Reference `refs/2_define_for_specify.md` for blog writing best practices
5. Include sections: Blog Type Declaration, Target Audience, Content Pillars, Publishing Cadence, Success Criteria, Distribution Requirements

**Reference**: See `refs/2_define_for_specify.md` for guidance on blog content strategy.
